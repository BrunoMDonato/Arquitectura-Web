# Gestión Inmobiliaria 

## Propósito
Este proyecto tiene como objetivo modelar una API para uso interno de una inmobiliaria.  
La API permitiría administrar propiedades y clientes, facilitando operaciones básicas como **altas, consultas, modificaciones y bajas (CRUD)**.

## Entidades principales
- **Propiedad**: representa una casa, departamento o local que se administra.  
- **Cliente**: persona interesada en alquilar o comprar una propiedad.  
- **Agente**: empleado de la inmobiliaria que gestiona propiedades y clientes.  

## Operaciones CRUD 
### Propiedades
- Crear una nueva propiedad  
- Consultar listado de propiedades  
- Actualizar precio o estado  
- Eliminar registro  

### Clientes
- Registrar un cliente  
- Consultar datos  
- Modificar información  
- Dar de baja  

### Agentes
- Registrar agentes  
- Consultar información  
- Actualizar datos  
- Eliminar registro
